## Architecture

Astro SSG site in French for dog insurance comparison. No framework components — pure `.astro` files.

### Layout hierarchy

```
Layout.astro          ← base HTML shell + ALL global CSS
├── RaceLayout.astro  ← individual race pages (31 pages)
├── ContentLayout.astro ← guides & editorial content
└── (direct)          ← comparatif, index, legal pages
```

**`Layout.astro`** contains the entire design system as one `<style is:global>` block — every reusable CSS class lives here (`.callout`, `.check-list`, `.data-table`, `.article-grid`, `.faq-list`, `.tier-card`, etc.). There is no external CSS file.

**`RaceLayout.astro`** has its own `<style is:global>` block for race-page-specific styles (`.metrics-row`, `.race-article`, `.faq-btn`, `.key-figures`, etc.). It orchestrates the fixed page sections: `HeroBreed` → `KeyFigures` → `RiskGauge` → `<slot>` (editorial content) → `InsuranceCards` → similar races grid → CTA banner.

### Race pages (`src/pages/races/*.astro`)

31 static pages. `labrador.astro` is the canonical reference. Each page:

1. Passes props to `RaceLayout`: `nom`, `slug`, `taille` (`petite`|`moyenne`|`grande`), `risque` (`faible`|`modere`|`eleve`), `score` (0–100), `kf` (4 key figures), `similaires` (up to 4 related races)
2. Renders `<div class="race-article">` as the slot with a fixed 7-section structure:
   - `<p class="intro-text">` — intro paragraph
   - `<h2>Coût annuel…</h2>` + `<div class="table-wrap"><div class="table-scroll"><table class="data-table">`
   - `<h2>Maladies fréquentes…</h2>` + disease table
   - `<h2>Coût des soins…</h2>` + cost table
   - `<div class="callout">` — race-specific financial risk (with info SVG icon + `<h3>Risque financier spécifique [prep] [nom]</h3>`)
   - `<h2>Conseils de prévention…</h2>` + `<ul class="check-list">` (each `<li>` contains the SVG checkmark `<span class="check-list-icon">`)
   - FAQ with `<div class="faq-item">` / `<button class="faq-btn">` / `<div class="faq-answer">` (JS accordion in RaceLayout)

TOC is auto-built from `h2` elements via IntersectionObserver JS in `RaceLayout.astro`.

### French preposition rules for race names

Used in `<h3>` titles and section headings:
- `"à l'"` — vowel-initial names: *Akita Inu*, *Épagneul Breton*
- `"à la"` — feminine: *Montagne des Pyrénées*
- `"au"` — all others

### CSS patterns (key classes from `Layout.astro`)

| Class | Purpose |
|---|---|
| `.data-table` | Styled table — light gray header, alternating rows |
| `.callout` | Yellow info box (grid `auto 1fr`) with `.callout-icon` |
| `.check-list` | SVG-checkmark list; each `<li>` needs `<span class="check-list-icon"><svg…>` |
| `.article-grid` | Two-column layout: 220px sticky TOC + `1fr` content |
| `.article-body` | Right column, max-width 760px |
| `.faq-list` | `<details>`/`<summary>` accordion (guide pages) |
| `.faq-btn` | JS-powered accordion (race pages, uses `aria-expanded`) |
| `.tier-card` | Insurance plan card; `.is-recommended` adds green border |
| `.metrics-row` | 4-column key-figures grid (rendered by `KeyFigures.astro`) |

### Images

Race photos at `/public/images/races/{slug}.webp`. Two slugs have accented filenames — an `imgMap` in `RaceLayout.astro` handles the mapping (`bouledogue-francais` → `bouledogue-français`, `montagne-des-pyrenees` → `montagne-des-Pyrénées`).

### Sitemap

Auto-generated by `@astrojs/sitemap` on every build. Priority/changefreq are configured via `serialize()` in `astro.config.mjs` — edit there, not in `dist/`.

### Batch scripts (`scripts/`)

Node.js ESM scripts for bulk edits across all 31 race pages:
- `harmonize-races.mjs` — removes inline `<style>` blocks, converts `section-prevention` → `check-list` with SVG icons
- `callout-risques.mjs` — wraps "Risques financiers" `h2+p` in `.callout` structure with correct French prepositions

Run with `node scripts/<name>.mjs` from the project root.
